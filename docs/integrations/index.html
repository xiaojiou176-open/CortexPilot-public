Boundary

No fake plugin claims

The truthful product story today is integration-by-protocol, integration-by-package, and integration-by-skill pack. It is not “install our official plugin from a store.”

What teams can actually adopt today

  • Command Tower as the single operator surface above coding-agent runs
  • Workflow Cases as the shared operating record that ties request, queue, verdict, and linked runs together
  • Proof & Replay as the review surface you use before trusting an outcome
  • read-only MCP when you want machine-readable inspection without mutation
  • frontend client + contract + shared UI substrate when you want repo-owned builder entrypoints
  • repo-local skills when you want coding agents to use the same operating playbook

How to talk about this truthfully

Good wording

“CortexPilot works with Codex and Claude Code workflows through Workflow Cases, Proof & Replay, read-only MCP, repo-owned skills, and builder entrypoints.”

Bad wording

“CortexPilot already ships an official Codex / Claude Code / OpenClaw plugin.”

Good wording

“OpenClaw stays in the adjacent coding-agent layer and can use CortexPilot as a bundle-compatible review surface when the mapped path is explicitly shipped and tested.”

Bad wording

“OpenClaw is a first-class officially supported plugin surface inside CortexPilot today.”
