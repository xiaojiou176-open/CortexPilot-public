API and contract quickstart

CortexPilot is not a full SDK platform today, but it already exposes a useful API and contract edge for dashboard, desktop, and custom frontend consumers. This page is the shortest truthful map.

1. OpenAPI document

Start with the public OpenAPI document if you want the machine-facing HTTP contract.

2. Thin frontend API client

Use @cortexpilot/frontend-api-client when you want convenience helpers over runs, Workflow Cases, approvals, and command-tower reads.

3. Contract-facing types

Use @cortexpilot/frontend-api-contract when you want route/query/type boundaries without importing backend modules.

4. Shared presentation substrate

Use @cortexpilot/frontend-shared when you need shared copy, locale helpers, and status presentation instead of rebuilding those primitives in each app.

Current boundary

  • useful builder edge today, not a full SDK platform
  • read-oriented integration story, not a hosted operator service
  • best read alongside the read-only MCP quickstart and the AI + MCP + API map
