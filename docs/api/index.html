API and contract quickstart

CortexPilot is not a full SDK platform today, but it already exposes a useful API and contract edge for dashboard, desktop, and custom frontend consumers. This page is the shortest truthful map.

Choose the shortest starting point

  • Stay on this page when you want the HTTP contract, thin client story, and contract-facing type boundary.
  • Jump to the read-only MCP quickstart when you need the protocol boundary first and package details later.
  • Jump to the builder quickstart when you are ready to choose the package surface for a frontend integration.

1. OpenAPI document

Start with the public OpenAPI document if you want the machine-facing HTTP contract.

2. Thin frontend API client

Use @cortexpilot/frontend-api-client when you want convenience helpers over runs, Workflow Cases, approvals, and command-tower reads.

External teams that want the shortest repo-owned starter should begin with createControlPlaneStarter(...) on top of that client, because it bundles the current overview + agents + contracts + role config bootstrap path without implying a hosted SDK platform.

The repo-owned starter example shows the same bootstrap + preview path as runnable code instead of leaving builders to reconstruct it from prose alone.

The current contract now also includes a queue-first operator mutation pilot shape: /api/queue/from-run/{run_id}/preview and /api/queue/{queue_id}/cancel exist as repo-owned HTTP control-plane routes. Treat them as guarded operator mutation surfaces, not as public write-capable MCP.

3. Contract-facing types

Use @cortexpilot/frontend-api-contract when you want route/query/type boundaries without importing backend modules.

4. Shared presentation substrate

Use @cortexpilot/frontend-shared when you need shared copy, locale helpers, and status presentation instead of rebuilding those primitives in each app.

Current boundary

  • useful builder edge today, not a full SDK platform
  • read-oriented integration story, not a hosted operator service
  • queue preview/cancel routes are repo-owned operator HTTP surfaces, not public write-capable MCP
  • best read alongside the read-only MCP quickstart and the AI + MCP + API map

Minimal onboarding order

  1. Read the OpenAPI document if you need the raw HTTP contract.
  2. Use the thin client package when you want convenience helpers over that contract.
  3. Use contract-facing types when you want typed route/query boundaries without importing backend modules.
  4. Pair the result with the builder quickstart so package-level adoption stays aligned with the current MCP/API story.
node packages/frontend-api-client/examples/control_plane_starter.local.mjs \
  --base-url http://127.0.0.1:10000 \
  --role WORKER \
  --mutation-role TECH_LEAD \
  --preview-provider cliproxyapi \
  --preview-model gpt-5.4

This example is preview-first by default. Use --apply only when you intentionally want to persist repo-owned role defaults through the same starter path.
